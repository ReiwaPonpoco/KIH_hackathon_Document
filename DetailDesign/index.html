バックエンド設計

Firebase Functionsを利用する

1. 周辺の情報を取得するエンドポイント

  • メソッド: GET
  • パス: /getPlaces
  • 入力: 緯度、経度
  • 動作:
    • Google Maps Places APIを呼び出し
    • 位置情報の周辺の情報を取得
  • 出力: 周辺の情報のリスト

2. コンテンツ翻訳エンドポイント

  • メソッド: POST
  • パス: /translateContent
  • 入力: 翻訳するコンテンツ、翻訳先の言語
  • 動作:
    • Google Cloud Translate APIを呼び出し
    • コンテンツを指定された言語に翻訳
  • 出力: 翻訳されたコンテンツ

3. 最適なルートを取得するエンドポイント

  • メソッド: GET/POST (具体的な方法による)
  • パス: /getOptimalRoute (または適切な名前)
  • 入力: 現在地、目的地
  • 動作:
    • 適切なAPIまたはライブラリを利用してルート情報を取得
  • 出力: 最適なルート情報

フロントエンド設計

index.html (ルートページ)

機能:

  • 言語の選択
  • ユーザ登録へのリンク
  • ログインへのリンク
処理:
  • 言語設定の保存 (localStorageなどを使用)
  • ページの翻訳

ar-app.html (ARアプリケーションのメインページ)

機能:

  • カメラ起動
  • ARコンテンツの表示
  • 現在地の表示
処理:
  • カメラのアクセス許可
  • ARコンテンツのレンダリング
  • ページの翻訳

signup.html (ユーザ登録ページ)

機能:

  • ユーザ情報の入力 (メールアドレス, パスワード)
  • 登録ボタン
処理:
  • Firebase Authenticationを利用したユーザ登録
  • ページの翻訳

login.html (ユーザログインページ)

機能:

  • メールアドレスとパスワードの入力
  • ログインボタン
処理:
  • Firebase Authenticationを利用したユーザログイン
  • ページの翻訳

home.html (ナビゲーションページ)

機能:

  • ARアプリケーション起動ボタン
  • お気に入りページへのリンク
処理:
  • ページの翻訳

favorite.html (お気に入りリストページ)

機能:

  • お気に入りの場所リスト表示
処理:
  • Firebaseからお気に入りデータの取得
  • ページの翻訳

navigation.html (ナビゲーションページ)

機能:

  • 現在地から選択したお気に入りの場所へのルート表示
処理:
  • Google Maps APIを利用したルート取得
  • ページの翻訳
